### 2026-03-02 — Vanilla JS без фреймворков
**Контекст:** Прототип Mini App для бьюти-мастера, нужно быстро собрать и проверить гипотезу.
**Варианты:** React + Vite (как другие проекты), Vue, Vanilla JS
**Решение:** Vanilla HTML + CSS + JS (ES Modules) — без фреймворков и бандлера.
**Последствия:** Мгновенная загрузка в Telegram (нет бандла), простой деплой (статика), но сложнее масштабировать. При росте проекта — миграция на React/Vite.

---

### 2026-03-02 — Vercel для всего (статика + бот)
**Контекст:** Нужен HTTPS для Mini App + обработчик webhook для бота.
**Варианты:** Vercel (статика), Vercel + VPS (бот отдельно), Cloudflare Pages
**Решение:** Vercel — статика + Serverless Functions (`api/webhook.js`) в одном проекте.
**Последствия:** Один деплой, один URL. Webhook бота живёт рядом с фронтендом. При холодном старте — задержка ~500ms, но для бота это приемлемо.

### 2026-03-02 — CSS ThemeParams вместо своей темы
**Контекст:** Mini App должен выглядеть нативно в Telegram (светлая и тёмная тема).
**Варианты:** Своя дизайн-система с двумя темами, CSS ThemeParams от Telegram
**Решение:** Все цвета берутся из `--tg-theme-*` CSS-переменных. В браузере — фоллбэки.
**Последствия:** Автоматическая поддержка тёмной темы, нативный вид. Но в браузере вне Telegram — только светлая тема с фоллбэк-цветами.

### 2026-03-02 — localStorage для состояния
**Контекст:** Нужно запоминать: показан ли оффер, пройдён ли welcome.
**Варианты:** localStorage, Telegram CloudStorage, серверная БД
**Решение:** localStorage — просто и работает. Два ключа: `offer_shown`, `welcome_shown`.
**Последствия:** Состояние привязано к браузеру/устройству, не к пользователю. При смене устройства — покажется снова. Для MVP это нормально; при подключении Supabase — перенести на сервер.
