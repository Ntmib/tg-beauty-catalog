# Тестирование — tg-app (Mini App)

> После каждого изменения проходим по чеклисту.
> Отмечаем [x] пройденные пункты, [!] — найденные баги.

## 0. Предварительные шаги
- [ ] Открыть http://localhost:8080 (или Vercel URL) — страница загрузилась
- [ ] Консоль браузера — нет ошибок (DevTools → Console)
- [ ] Переключатель «Клиент / Мастер» работает

## 1. Клиент — Каталог (catalog.js)
- [ ] Профиль мастера: имя, специализация, адрес отображаются
- [ ] Портфолио: сетка 3×2 с фото
- [ ] Услуги: список с ценами и длительностью
- [ ] Клик по услуге → экран К3
- [ ] Клик по фото → лайтбокс К2
- [ ] Кнопка «Мои записи» → экран К6
- [ ] Кнопка «Поделиться с другом» → открывает Telegram шеринг
- [ ] Кнопки «Позвонить» и «Написать» работают

## 2. Клиент — Детали услуги (service.js)
- [ ] Название, цена, длительность, описание отображаются
- [ ] Фото в горизонтальном скролле
- [ ] Кнопка «← Назад» → возврат в каталог
- [ ] Кнопка «Записаться» → экран К4

## 3. Клиент — Запись (booking.js)
- [ ] Даты: горизонтальный скролл, 14 дней вперёд
- [ ] Нерабочие дни отмечены как disabled
- [ ] Выбор даты → появляются слоты времени
- [ ] Занятые слоты — disabled
- [ ] Выбор времени → появляется итог записи
- [ ] Кнопка «Записаться» → экран К5

## 4. Клиент — Успех (success.js)
- [ ] Анимированная галочка
- [ ] Детали записи (дата, время, услуга)
- [ ] Кнопка «Мои записи» → К6
- [ ] Ссылка «Вернуться в каталог» → К1

## 5. Клиент — Мои записи (records.js)
- [ ] Разделение на предстоящие / прошлые
- [ ] Кнопки: отменить, перенести, повторить
- [ ] Пустое состояние — корректное

## 6. Модалка-оффер (offer.js)
- [ ] Появляется при первом открытии
- [ ] Кнопка «Получить скидку 15%» → t.me/Beauty_100master_bot?start=from_app
- [ ] Кнопка «Пропустить» → закрывает
- [ ] Повторно не появляется (localStorage)
- [ ] В режиме мастера — не появляется

## 7. Экран приветствия (welcome.js)
- [ ] Имя пользователя из Telegram (или «Тест» в браузере)
- [ ] 3 пункта — услуги, запись, напоминания
- [ ] Кнопка «Начать» → каталог
- [ ] Повторно не появляется (localStorage)

## 8. Мастер — Дашборд (dashboard.js)
- [ ] Ссылка на бота + кнопка копирования
- [ ] Ближайшие записи с кнопками подтвердить/отклонить
- [ ] Сетка 2×2: услуги, фото, расписание, профиль
- [ ] «Посмотреть как клиент» → каталог

## 9. Мастер — Услуги (master-services.js, service-edit.js)
- [ ] Список услуг с переходом в редактирование
- [ ] Форма: название, цена, длительность, описание
- [ ] Кнопка сохранить

## 10. Мастер — Портфолио (master-portfolio.js)
- [ ] Сетка фото с кнопками удаления
- [ ] Кнопка «Добавить фото»
- [ ] Счётчик X из 20

## 11. Мастер — Расписание (master-schedule.js)
- [ ] Тогглы рабочих дней
- [ ] Выбор часов работы и перерыва
- [ ] Тоггл «Не работаю завтра»

## 12. Мастер — Записи (master-bookings.js)
- [ ] Табы: Новые / Все / Прошлые
- [ ] Кнопки подтвердить / отклонить
- [ ] Группировка по дате

## 13. Мастер — Профиль (master-profile.js)
- [ ] Поля: имя, специализация, адрес, телефон, telegram, опыт
- [ ] Мультивыбор специализаций (chips)

## 14. Webhook бота (api/webhook.js)
- [ ] GET /api/webhook → `{"ok": true}`
- [ ] /start → приветствие + кнопка «Открыть каталог»
- [ ] /help → инструкция + контакт мастера
- [ ] /start from_app → промокод BEAUTY15
- [ ] Любое сообщение → подсказка

## 15. Негативные сценарии
- [ ] Несуществующая услуга — показывает «Услуга не найдена»
- [ ] Пустые записи — показывает empty state
- [ ] Отсутствие Telegram SDK (браузер) — фоллбэки работают
- [ ] Двойной клик по кнопке — не ломает навигацию
