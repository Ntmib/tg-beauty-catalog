## 1. Анализ конкурентов и аналогов

### 1.1 DIKIDI (dikidi.net)

**Что делает:** Крупнейшая CRM для бьюти-индустрии в СНГ. Онлайн-запись + маркетплейс мастеров.

**Сильные стороны:**
- Бесплатный старт для одиночных мастеров (порог входа = 0)
- Маркетплейс — клиенты находят мастеров через приложение DIKIDI
- Мобильное приложение для мастера
- Простой интерфейс (проще чем YCLIENTS)
- Телеграм-бот для уведомлений

**Слабые стороны:**
- Бесплатная версия показывает рекламу КОНКУРЕНТОВ на странице записи мастера
- Маркетплейс — палка о двух концах: клиент может уйти к другому мастеру
- Нет Telegram Mini App — бот только шлёт уведомления, запись ведёт на веб
- Ограниченная кастомизация — не ощущается как "свой" каталог
- Дизайн страницы записи устаревший

**Что мы можем сделать лучше:**
- Каталог выглядит как личная визитка мастера (без чужой рекламы)
- Нативный Telegram Mini App вместо веб-переходов
- Красивый визуальный каталог с фото работ, а не скучная таблица услуг

---

### 1.2 YCLIENTS (yclients.com)

**Что делает:** Лидер рынка CRM для услуг в СНГ. Полный набор: запись, CRM, склад, финансы, аналитика.

**Сильные стороны:**
- Полная бизнес-автоматизация (CRM, складской учёт, финансы)
- Огромная клиентская база (де-факто стандарт для крупных салонов)
- Интеграции с кассами (54-ФЗ), IP-телефонией
- Мощная аналитика и отчёты
- Маркетплейс — клиенты находят салоны через приложение

**Слабые стороны:**
- Избыточно для одиночного мастера — 90% функций не нужны
- Дорого: от 1500 руб/мес для одного мастера, 3500-5000 руб/мес для салона
- Долгая и сложная настройка
- Нет Telegram Mini App — только веб-виджет
- Интерфейс перегружен функциями

**Что мы можем сделать лучше:**
- Настройка за 5 минут (а не за дни)
- Только то, что нужно: каталог + портфолио + запись + контакты
- Цена в 3-5 раз ниже (300-500 руб/мес vs 1500+)
- Нативная работа внутри Telegram — без переходов на внешние сайты

### 1.3 Salebot (salebot.pro)

**Что делает:** Конструктор Telegram-ботов с CRM и поддержкой Mini App. Один из немногих no-code инструментов, реально создающих Mini App.

**Сильные стороны:**
- Можно создать Telegram Mini App без кода
- Встроенная CRM и приём оплаты
- Относительно доступная цена (от 799 руб/мес)
- Гибкий визуальный конструктор сценариев

**Слабые стороны:**
- Mini App шаблоны выглядят шаблонно — "магазинный" вид без индивидуальности
- Не заточен под бьюти: нет расписания мастера, слотов, портфолио работ
- Крутая кривая обучения — сам конструктор сложен для не-технаря
- Для красивого каталога с фото работ придётся долго возиться
- UI конструктора перегружен

**Что мы можем сделать лучше:**
- Заточенность именно под бьюти (портфолио до/после, услуги, расписание)
- Красивый визуал "из коробки" — мастер только загружает фото и цены
- Настройка за 5 минут без изучения конструктора

### 1.4 Rubitime (rubitime.com)

**Что делает:** Сервис онлайн-записи с полноценным Telegram Mini App — одно из немногих решений с реальным Mini App для записи.

**Сильные стороны:**
- Реальный Telegram Mini App с визуальным интерфейсом записи
- Пошаговый флоу: услуга -> мастер -> дата/время
- Интеграция с Google Calendar
- Автоматические напоминания
- Бесплатный тариф

**Слабые стороны:**
- Универсальный (не заточен под бьюти)
- Нет портфолио мастеров и фото работ
- Дизайн функциональный, но без визуального лоска
- Нет галереи "до/после", нет социального доказательства (отзывы)

**Что мы можем сделать лучше:**
- Фокус на визуале (бьюти = эстетика, фото работ — ключевой элемент)
- Портфолио мастера как первый экран (не скучное расписание)
- Мультимастерная SaaS-модель (каждый мастер — свой каталог)

### 1.5 Кастомные боты от фрилансеров

**Что делает:** Огромный сегмент: частные мастера заказывают у фрилансеров ботов для записи (5000-50000 руб разово).

**Сильные стороны:**
- Делается "под себя"
- Одноразовая оплата (дёшево на старте)
- Простая логика (кнопки: "Услуги", "Записаться", "Контакты")

**Слабые стороны:**
- Выглядит как текстовый бот 2020 года — кнопки + текст, нет визуала
- Нет поддержки и обновлений после сдачи
- Зависимость от конкретного разработчика
- Нет красивого каталога с фото — только текст
- Не масштабируется
- Хостинг нужно оплачивать и поддерживать самостоятельно

**Что мы можем сделать лучше:**
- Красивый визуальный Mini App вместо текстовых кнопок
- Не нужен разработчик — мастер настраивает сам
- Обновления, поддержка, новые функции в рамках подписки
- Портфолио с фото, а не текстовый список услуг

## 3. Примеры отличных Mini Apps (из других ниш)

### Telegram Wallet
Криптокошелёк от Telegram. Эталон качества: плавные анимации, нативный UI, безупречная работа с MainButton. Показывает, что Mini App может выглядеть как полноценное нативное приложение.

### Telegram Gifts
Встроенная система подарков. Красивые карточки с Lottie-анимациями, категории, фильтрация. Отличный пример каталога — именно так должен выглядеть каталог услуг.

### Durger King (@shop)
Демо-бот от Telegram. Каноничная реализация каталога с оплатой: список → карточка → корзина → оплата. Простая структура, правильные паттерны.

### Hamster Kombat / Notcoin
Десятки миллионов пользователей. Показали силу реферальной системы и вирусности через Telegram. Урок: "Пригласи подругу — получи скидку" может сильно работать.

## 4. Типичные экраны Mini App каталога услуг

### Минимум для MVP (6 экранов):

**1. Главная / Витрина**
- Шапка: логотип мастера, имя, специализация, рейтинг
- Горизонтальная карусель акций / фото лучших работ
- Категории услуг (сетка иконок 2x2)
- Быстрая запись (1-3 самых популярных услуги)
- Нижний таб-бар: Каталог / Портфолио / Записи / Профиль

**2. Каталог услуг**
- Табы/chips категорий вверху (Волосы, Ногти, Лицо, Тело)
- Карточки услуг: фото, название, длительность, цена
- Возможность выбрать несколько услуг
- Строка поиска (при количестве услуг > 10)
- MainButton: "Записаться" (с badge "Выбрано: 2")

**3. Детали услуги**
- Галерея фото работ (до/после)
- Полное описание, длительность, цена
- Список мастеров, выполняющих услугу
- Отзывы клиентов
- MainButton: "Записаться"

**4. Выбор даты и времени**
- Горизонтальный скролл дат (14-30 дней вперёд)
- Сетка свободных слотов (кнопки 09:00, 09:30, 10:00...)
- Занятые слоты — серые/неактивные
- Сводка: "15 марта, 14:00 / Маникюр / Мастер Анна"
- MainButton: "Подтвердить запись"

**5. Подтверждение записи**
- Сводка: услуга, мастер, дата, время, стоимость
- Поле для комментария ("Есть пожелания?")
- Контакт подтянут из Telegram автоматически
- MainButton: "Подтвердить" или "Оплатить"

**6. Мои записи**
- Табы: "Предстоящие" / "Прошедшие"
- Карточка: услуга, мастер, дата, статус
- Кнопки: "Отменить", "Перенести", "Повторить запись"

### Дополнительные экраны (полная версия):

- **Профиль мастера** — фото, портфолио работ, отзывы, услуги с ценами
- **Экран успеха** — анимация, "Добавить в календарь", "Поделиться", "Записать ещё"
- **Портфолио / Галерея** — фото работ сеткой (как Instagram)
- **Профиль клиента** — история записей, избранное, уведомления

## 6. Telegram-специфичные требования

### Обязательно использовать:

| API | Зачем |
|-----|-------|
| `MainButton` | CTA-кнопка "Записаться" / "Подтвердить" |
| `BackButton` | Навигация назад (на всех экранах кроме главного) |
| `ThemeParams` | Адаптация к светлой/тёмной теме пользователя |
| `HapticFeedback` | Вибрация при выборе слота, подтверждении, ошибке |
| `expand()` | Развернуть на полный экран при открытии |
| `disableVerticalSwipes()` | Не закрывать при скролле каталога вниз |
| `initData` | Авторизация без логина (имя, id из Telegram) |
| `requestWriteAccess()` | Для отправки напоминаний через бота |

### Рекомендовано:

| API | Зачем |
|-----|-------|
| `CloudStorage` | Избранные мастера, история просмотров |
| `showPopup()` / `showConfirm()` | Нативные диалоги для подтверждений |
| `shareToStory()` | "Записалась на маникюр!" в Stories |
| `openInvoice()` | Оплата предзаписи |
| Direct Links (`t.me/bot?startapp=service_123`) | Ссылка на конкретную услугу |
| `enableClosingConfirmation()` | Не терять корзину при случайном закрытии |
| `addToHomeScreen()` | Добавить на домашний экран |

## 7. Частые ошибки при создании Mini Apps

### UX-ошибки:
- Не вызвать `disableVerticalSwipes()` — пользователь скроллит каталог, а Mini App закрывается
- Хардкодить цвета — в тёмной теме приложение слепит белым
- Кастомная кнопка "Назад" вместо нативного `BackButton`
- Нет skeleton loading — белый экран 2-3 секунды
- Слишком много шагов до записи (цель: максимум 4-5 тапов)

### Технические ошибки:
- Не проверять `initData` на сервере (дыра в безопасности)
- Большой bundle (> 200KB gzip) — долгая загрузка
- Не использовать `CloudStorage` — потеря данных при переоткрытии
- Не обрабатывать viewport при открытии клавиатуры
- Изображения без оптимизации (WebP, lazy loading, blur placeholder)

## 9. Где пустая ниша — наше позиционирование

### Что никто не делает:

1. **Красивый Telegram Mini App именно для бьюти-мастера** — с фотогалереей работ, каталогом, записью. Чтобы выглядел как Instagram-профиль, но внутри Telegram.

2. **Простой старт за 5 минут** — YCLIENTS требует дни настройки. DIKIDI проще, но ведёт на свою платформу. Конструкторы ботов требуют строить сценарии.

3. **"Свой бренд" в Telegram** — DIKIDI показывает рекламу конкурентов. YCLIENTS уводит на свой сайт. Ни одна платформа не даёт мастеру ощущение "это моё мини-приложение".

4. **Каталог работ с фото как главный элемент** — все фокусируются на записи (CRM-подход). Никто не делает визуальный каталог-портфолио как первый экран.

5. **Нативная Telegram Mini App** — 95% решений это либо веб-виджет, либо текстовый бот с кнопками. Mini App с нативным ощущением в бьюти-нише не делает практически никто.

### Наш проект — это:

- **Не CRM** (проще чем YCLIENTS/DIKIDI)
- **Не конструктор ботов** (красивее чем Salebot)
- **Не маркетплейс** (нет рекламы конкурентов)
- **Именно Mini App** — нативное ощущение, фото-каталог, запись, всё внутри Telegram
- **SaaS для мастеров** — каждый мастер настраивает под себя за 5 минут и получает личный каталог

### Ключевое отличие:

> Бьюти — это про эстетику. Если инструмент мастера выглядит уродливо, он его не будет использовать. Мы делаем красивый каталог, которым мастер будет гордиться и показывать клиентам.

## 10. Пользовательские сценарии

### Сценарий A: Клиент записывается (основной)
```
Клиент получает ссылку от мастера → Открывает бота → Жмёт "Открыть каталог"
→ Mini App: видит фото работ и услуги → Выбирает услугу
→ Выбирает дату/время → Подтверждает → Бот шлёт подтверждение
→ За 24ч и 2ч — бот шлёт напоминание
```

### Сценарий B: Мастер настраивает каталог
```
Мастер открывает бота → Авторизуется как мастер (через Telegram)
→ Админ-панель (Mini App): добавляет услуги, фото, цены, расписание
→ Получает ссылку на свой каталог → Делится с клиентами
→ Получает уведомления о новых записях
```

### Сценарий C: Повторная запись
```
Клиент открывает Mini App → "Мои записи" → "Повторить"
→ Услуга и мастер предзаполнены → Выбирает время → Готово
```

### Сценарий D: Вирусное распространение
```
Клиент записался → "Поделиться в Stories" → Фото работы + ссылка на мастера
→ Подруга видит → Переходит по ссылке → Открывает каталог → Записывается
```

## 12. Экспертная оценка

### 12.1 Взгляд UX-дизайнера

#### Главная проблема, которую нужно решить первой

У тебя ДВА совершенно разных пользователя с разными задачами:
- **Мастер** — настраивает каталог (загружает фото, пишет цены, задаёт расписание)
- **Клиент** — листает каталог и записывается

Это два разных интерфейса в одном приложении. И именно здесь новички ломаются — пытаются запихнуть и админку, и витрину в один экран.

**Рекомендация:** Один бот, но два режима Mini App:
- `t.me/bot?startapp=master` — админ-панель мастера (настройка каталога)
- `t.me/bot?startapp=catalog_ID` — каталог для клиента (конкретного мастера)

Или ещё проще: мастер управляет через отдельный раздел в том же Mini App (переключатель роли по `initData.user.id`). Клиент вообще не видит админские функции.

#### Ошибки, которые делают новички

**1. Перегруженный онбординг мастера**
Мастер маникюра — не технарь. Она откроет приложение, увидит 15 полей для заполнения и закроет. Навсегда.

Правильно: пошаговый онбординг из 3-4 экранов:
- Шаг 1: Имя + фото (подтянуть из Telegram автоматически) + специализация (выбор из списка: ногти / брови / ресницы / волосы / лицо)
- Шаг 2: Добавь первую услугу (название + цена + длительность, всё остальное — потом)
- Шаг 3: Выбери рабочие дни и часы (простой тоггл по дням недели)
- Готово. Каталог уже работает. Фото, описания, портфолио — добавишь позже

**2. Слишком "красивый" дизайн за счёт скорости**
Бьюти-мастера хотят красиво — но клиент хочет быстро. Если каталог грузится 3 секунды из-за 20 фото в высоком разрешении — клиент уйдёт.

Правильно:
- Thumbnail для списка (200x200, WebP, ~15KB)
- Полное фото только при открытии карточки (lazy load)
- Skeleton-заглушки пока фото грузятся
- Blur-placeholder (tiny base64 превью ~300 байт) пока грузится основное фото

**3. Универсальность ≠ сложность**
"Подходит для любого мастера" не значит "100 настроек". Это значит — умные дефолты.

Правильно:
- Для бровиста — автоматически предложить категории: "Коррекция", "Окрашивание", "Ламинирование"
- Для мастера ногтей — "Маникюр", "Педикюр", "Наращивание", "Дизайн"
- Мастер выбирает специализацию на первом шаге → получает преднастроенный шаблон с типичными услугами → редактирует под себя

Не пустая форма "введите название услуги", а готовый шаблон, который нужно только подправить.

**4. Игнорирование "пустых" состояний**
Мастер только создал каталог — в нём 0 фото, 1 услуга, нет отзывов. Клиент открывает — и видит пустоту. Доверие = 0.

Правильно:
- Показывать подсказки вместо пустоты: "Добавьте фото работ — клиенты записываются на 40% чаще"
- Скрывать блоки, в которых нет контента (нет портфолио — нет вкладки "Портфолио")
- Минимальный порог для публикации: хотя бы 1 услуга с ценой + фото мастера

#### Что обязательно должно быть (с точки зрения UX)

**Для мастера (админка):**
- Предпросмотр каталога "глазами клиента" (кнопка "Посмотреть как клиент")
- Уведомления о записях прямо в Telegram-чат (не в Mini App — мастер не будет держать его открытым)
- Редактирование расписания за 2 тапа (не календарь со слотами — а просто "Пн-Пт, 10:00-19:00, обед 13:00-14:00")
- Быстрое закрытие дня: "Завтра не работаю" — один тап
- Подтверждение записи вручную ИЛИ автоматически (мастер выбирает режим)

**Для клиента (каталог):**
- Открывается за < 1 секунду (first meaningful paint)
- Максимум 3 тапа от открытия до записи
- Телефон и Telegram — кликабельные (позвонить / написать)
- Адрес — кликабельный (открывает карту)
- Навигация без тупиков (BackButton на каждом экране)

#### Что бесит пользователей (клиентов) в подобных приложениях

| Раздражитель | Как у нас должно быть |
|-------------|----------------------|
| "Цена по запросу" / "от 1000р" без верхней границы | Точная цена. Или вилка "1000-1500р" с пояснением от чего зависит |
| Нет свободных дат, но непонятно когда есть | Показать ближайшее свободное время: "Ближайшее: среда, 14:00" |
| Записался — и тишина. Подтвердили? Нет? | Мгновенный статус: "Ожидает подтверждения мастера" → "Подтверждено" (push в чат) |
| Нельзя отменить без звонка | Кнопка "Отменить" с подтверждением. Если < 24ч — предупреждение |
| Приложение выглядит одинаково для всех | Фото мастера, имя, стиль — с первого экрана понятно, к КОМУ ты записываешься |
| Надо регистрироваться / вводить телефон | Всё из Telegram автоматически. Ноль полей для заполнения |
| Непонятно, где я нахожусь (какой экран) | Таб-бар + BackButton + заголовок экрана |
| Мелкие кнопки, промахиваешься пальцем | Минимальный тап-таргет 44x44px. Слоты времени — крупные кнопки с отступами |
| Загрузка крутится вечно | Skeleton-лоадеры, оптимистичные обновления (показать результат до ответа сервера) |

### 12.2 Взгляд маркетолога

#### Что заставляет человека записаться (а не просто посмотреть и уйти)

**1. Фото работ — это твоё оружие номер один**

В бьюти клиент покупает глазами. Не текст "Маникюр классический — 1500р", а фотографию красивых ногтей. Причём не стоковую, а реальную работу конкретного мастера.

Рекомендации:
- Фото работ — ПЕРВОЕ, что видит клиент (не список услуг, не расписание)
- Главный экран = мини-Instagram: сетка красивых фото 3x3
- Тапнул на фото → видит какая это услуга и цену → "Записаться"
- Фото "до/после" — убийственный конвертер (особенно брови, ресницы, волосы)

**2. Социальное доказательство**

Клиент не знает этого мастера. Ему нужны сигналы доверия:
- Количество записей: "Записались 247 клиентов" (даже просто число)
- Отзывы (хотя бы 3-5 текстовых)
- "Постоянные клиенты" — блок с числом
- Опыт работы: "Мастер маникюра, 5 лет опыта"

Для MVP достаточно: число записей (считается автоматически) + опыт (мастер указывает). Отзывы — во второй версии.

**3. Срочность и ограниченность**

Простые триггеры, которые подталкивают к записи:
- "Осталось 2 свободных слота на эту неделю" (реальные данные, не фейк)
- "Ближайшее свободное время: завтра, 14:00" — конкретика вместо абстракции
- Если мастер сделал акцию — бейдж "Акция" на услуге
- "Запись на ближайшие 3 дня заполнена на 80%" — показать популярность

**4. Минимум трения**

Каждый лишний шаг — это потеря 20-30% людей. Воронка должна быть:
```
Открыл → Увидел фото → Тапнул услугу → Выбрал время → Подтвердил
     (5 касаний, 0 полей для заполнения)
```

Никаких:
- Регистраций (данные из Telegram)
- "Введите номер телефона" (он есть в Telegram, если мастер попросит — `requestContact()`)
- "Выберите филиал" (у частного мастера один адрес)
- "Согласен с политикой конфиденциальности" (убрать в подвал мелким шрифтом)

**5. Повторные записи — главный источник дохода мастера**

80% выручки бьюти-мастера — это постоянные клиенты. Приложение должно работать на удержание:
- "Повторить запись" — одна кнопка в "Мои записи"
- Бот напоминает через 3-4 недели: "Прошёл месяц с последнего маникюра. Записаться снова?"
- "Обычно вы записываетесь на 'Маникюр + покрытие'. Записать на привычное время?"

**6. Вирусность — встроенная, не навязанная**

Не "Пригласи 5 друзей и получи скидку" (это раздражает), а:
- После записи: "Поделиться в Stories" с красивой карточкой (фото работы + имя мастера + ссылка)
- Deep link: мастер скидывает клиенту ссылку `t.me/bot?startapp=m_123` — и клиент сразу видит ЕГО каталог
- Мастер шарит ссылку на конкретную услугу в своём Telegram-канале / Instagram

#### Элементы, которые повышают конверсию (по приоритету)

| # | Элемент | Влияние на конверсию | Сложность |
|---|---------|---------------------|-----------|
| 1 | Фото работ как главный экран | Критическое | Низкая |
| 2 | Запись без регистрации (данные из TG) | Критическое | Низкая |
| 3 | Точные цены (не "от ...") | Высокое | Низкая |
| 4 | "Ближайшее свободное время" на главной | Высокое | Средняя |
| 5 | Напоминания перед визитом (24ч, 2ч) | Высокое (снижает no-show) | Средняя |
| 6 | "Повторить запись" для постоянных | Высокое | Низкая |
| 7 | Число записей / опыт мастера | Среднее | Низкая |
| 8 | Фото до/после | Среднее | Низкая |
| 9 | Реферальная ссылка / шаринг | Среднее | Средняя |
| 10 | Отзывы клиентов | Среднее | Высокая (модерация) |

#### Критический момент для SaaS-модели

Ты продаёшь не клиентам, а МАСТЕРАМ. Мастер — твой настоящий покупатель. И мастер думает не "какой красивый каталог", а:
- "Мне это принесёт больше клиентов?"
- "Сколько времени на настройку?" (ответ должен быть: 5 минут)
- "Чем это лучше чем просто пост в Instagram?"

Ответ на последний вопрос и есть твоё УТП: **клиент записывается прямо из Telegram, не выходя в браузер. Ты получаешь уведомление. Клиент получает напоминание. Никаких "напишите в директ".**

### 12.3 Взгляд Telegram-разработчика

#### Что реально работает хорошо

**1. Авторизация — бесплатная и моментальная**
`initData` даёт тебе id, имя, фото, язык пользователя — без регистрации. Для каталога этого достаточно. Не нужны никакие формы логина.

На сервере — обязательно проверяй `initData` через HMAC-SHA-256 (bot_token как ключ). Это не опция, а требование безопасности. Без проверки любой может подделать запрос.

**2. MainButton — мощнее, чем кажется**
- Меняй текст под контекст: "Выбрать услугу" → "Выбрать время" → "Подтвердить запись"
- `showProgress()` — индикатор загрузки прямо в кнопке (пока сервер обрабатывает запись)
- Цвет автоматически подхватывается из темы
- Это единственная кнопка, которая ВСЕГДА видна — используй только для главного действия

**3. Уведомления через бота — лучше чем push**
Telegram-сообщение от бота видят 90%+ пользователей (vs 40-60% для push). Для напоминаний о записи — идеально. Но нужен `requestWriteAccess()` — запросить разрешение один раз.

**4. Deep Links — бесплатный маркетинговый инструмент**
`t.me/bot?startapp=master_123` — прямая ссылка на каталог конкретного мастера. Мастер кидает её в Instagram, WhatsApp, визитку — клиент тапает и сразу видит каталог. Параметр `startapp` может нести ID мастера, ID услуги и что угодно.

#### Ограничения, которые нельзя обойти

**1. Один бот = один Mini App**
Нельзя сделать несколько разных Mini App для одного бота. Но можно: один Mini App с роутингом, который по параметру `startapp` показывает разный контент (каталог мастера X или мастера Y).

**2. `sendData()` закрывает Mini App**
Если используешь `sendData()` для отправки данных боту — Mini App закроется. Для записи лучше отправлять данные через свой API (fetch на backend), а не через `sendData()`. Тогда Mini App остаётся открытым и можно показать экран успеха.

**3. Размер изображений — узкое место**
Telegram не хостит файлы Mini App. Фото хранятся на твоём бэкенде (Supabase Storage). Если мастер загрузит 50 фото по 5MB — каталог будет тормозить.

Решение:
- При загрузке — автоматическое сжатие (Supabase Edge Function или Vercel Image Optimization)
- Хранить 3 размера: thumbnail (200px), medium (600px), full (1200px)
- В каталоге показывать thumbnail, при открытии — medium
- Лимит на загрузку: 5MB на файл, конвертация в WebP

**4. Оффлайн не работает**
Mini App = web-страница. Нет интернета — нет каталога. Service Worker поможет кэшировать структуру, но свежие данные (расписание, слоты) всё равно нужно грузить. Не обещай "работает без интернета".

**5. iOS-ограничения**
- На iOS `HapticFeedback` работает не на всех моделях
- `addToHomeScreen()` может не работать на старых iOS
- Виртуальная клавиатура на iOS сдвигает viewport — нужно обрабатывать через `window.visualViewport`
- Высота Mini App на iOS может быть меньше, чем на Android — тестируй на обоих

**6. Платежи — сложная тема для MVP**
Telegram Payments требуют подключения платёжного провайдера (ЮKassa и т.д.) + юридическое оформление. Telegram Stars — только для цифровых товаров.

Рекомендация для MVP: **не делать онлайн-оплату**. Просто запись + оплата на месте. Предоплату добавить во второй версии, когда будет юрлицо и подключённый провайдер.

#### Что лучше НЕ делать (чтобы не усложнять)

| Не делать в MVP | Почему | Когда добавить |
|----------------|--------|----------------|
| Онлайн-оплату | Юрлицо + провайдер + чеки. Для MVP — оверкилл | v2 |
| Систему отзывов | Модерация, фейки, споры. Сложно сделать правильно | v2 |
| Полноценный календарь-планер для мастера | Огромная сложность синхронизации. Google Calendar API + конфликты | v2 |
| Поддержку нескольких мастеров в одном каталоге (салон) | Мультитенант-архитектура, расписания, роли. Другой уровень | v3 |
| Программу лояльности / бонусы | Сложная бизнес-логика | v3 |
| Интеграцию с Instagram | API Instagram ненадёжен, постоянно меняется | Никогда (или v3) |
| FullScreen mode | Скрывает хедер Telegram — пользователь теряется. Для каталога не нужен | Не нужно |
| Кастомные анимации на каждый чих | Тормозит на слабых телефонах, увеличивает bundle | Точечно |

#### Что делать обязательно с первого дня

```
1. disableVerticalSwipes() — иначе каталог нельзя скроллить
2. expand() — иначе открывается в половину экрана
3. ThemeParams через CSS-переменные — иначе тёмная тема сломана
4. BackButton на каждом экране кроме главного
5. Проверка initData на сервере — иначе дыра в безопасности
6. Оптимизация изображений — иначе каталог тормозит
7. requestWriteAccess() при первой записи — иначе напоминания не работают
```

#### Архитектура для мультитенантности (один бот — много мастеров)

Это ключевой технический вызов проекта. Один бот обслуживает тысячи мастеров, каждый со своим каталогом.

```
Структура URL:
  t.me/beautybot?startapp=m_ABC123    → каталог мастера ABC123
  t.me/beautybot?startapp=s_XYZ789    → услуга XYZ789
  t.me/beautybot?startapp=admin       → админка (если user.id совпадает с мастером)

Структура БД (Supabase):
  masters        → id, telegram_id, name, photo, specialty, schedule, settings
  services       → id, master_id, title, price, duration, category, photo_urls
  portfolio      → id, master_id, photo_url, service_id, description
  bookings       → id, master_id, client_telegram_id, service_id, date, time, status
  clients        → telegram_id, first_name, last_name, phone (из requestContact)
```

Ключевой принцип: `startapp` параметр определяет, чей каталог показать. Бот один, а каталогов — столько, сколько мастеров.

### 12.4 Итоговые рекомендации — что делать прямо сейчас

#### MVP (версия 1.0) — минимум, который уже имеет ценность

**Для мастера:**
- Пошаговый онбординг (имя + фото из TG + специализация → первая услуга → расписание)
- Шаблоны услуг по специализации (не пустая форма)
- Загрузка фото работ (до 20 фото, автосжатие)
- Простое расписание (рабочие дни + часы, кнопка "не работаю завтра")
- Уведомления о записях в Telegram-чат
- Предпросмотр каталога глазами клиента
- Ссылка на свой каталог для шаринга

**Для клиента:**
- Визуальный каталог: фото работ → услуги с ценами → запись
- Запись за 3 тапа без регистрации
- Выбор даты/времени из свободных слотов
- Подтверждение + напоминания (24ч, 2ч)
- "Мои записи" с возможностью отмены
- Кликабельный адрес и контакты мастера

**НЕ делать в MVP:**
- Онлайн-оплату
- Отзывы
- Реферальную систему
- Мультимастерные каталоги (салоны)
- Аналитику для мастера
- Программу лояльности

#### Метрики успеха MVP

| Метрика | Целевое значение |
|---------|-----------------|
| Время онбординга мастера | < 5 минут |
| Тапов до записи (клиент) | 3-4 |
| First Meaningful Paint | < 1.5 секунды |
| Bundle size (gzip) | < 150KB |
| Конверсия "открыл → записался" | > 15% |
| No-show (не пришёл) | < 15% (за счёт напоминаний) |
